Q-2. What is a Bootstrap Container, and how does it work?

Answer:
A container is the most basic layout component in Bootstrap. Bootstrap Containers are basic bootstrap building components that enclose a page's content. It is responsible for arranging and aligning content within it in accordance with the viewport or individual device. Container definitions can be found in the container class (.container). In other words, the width of the containers influences how much content appears in the layout. Additional components and material are added to the container.
There are many uses for containers, It needs to be used with the standard grid system, to determine the layout's width and provide web content, for any web project to have responsive fixed behaviour.
Basically, bootstrap offers two different kinds of container classes:
i.Default-Container: The '.container' class is used by the default container. It offers a fixed-width container that is responsive
syntax:
<div class="container">
<!-- Content here -->
</div>
ii. Fluid-Container: The '.container-fluid' class is used by fluid-container. It is utilised for a container that fills the viewport's entire width.
syntax:
<div class="fluid-container">
<!-- Content here -->
</div>

Q-4. What do you know about the Bootstrap Grid System?

Answer:
The grid system in Bootstrap allows up to 12 columns to span the width of the page. You can use each one individually or combine them to make larger columns. Any value combination that adds up to 12 may be utilised. For each, use 12 columns of width 1, or 4 columns of width 3, or any other combination
Grid Classes: grid system features four classes that can be combined to create layouts with additional flexibility:
  • xs (<576px): For Portrait Mobile Phones.
  • sm (>=576px): For Landscapes phones
  • md (>=768px): For Tablets/Phablets
  • lg (>=992px): For Small-sized Desktops/Laptops
  • xl (>=1200px): For Larger-sized Desktops/Laptops

Note: Depending on the size of your screen, the output may vary slightly from what is seen.
